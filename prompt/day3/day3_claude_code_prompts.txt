주제: 제조 AI Agent 아키텍처 설계 관점의 Claude Code 활용

============================================================
실습 목적
============================================================

이 실습의 목적은 Claude Code로 코드를 자동 생성하는 것이 아닙니다.

Claude Code를 사용하여 src/day3 폴더의 실습 파일들이 전체 Agent 아키텍처에서 어떤 역할을 하는지 빠르게 파악하고, Tool 분할, Tool Description, State 전달, 실행 기록 관점에서 설계 리뷰를 수행합니다.

Claude Code의 답변은 최종 정답이 아니라 설계 리뷰 초안입니다.
최종 판단은 사람이 해야 합니다.

============================================================
실습 전 주의사항
============================================================

1. 바로 코드를 수정하지 않습니다.
2. 먼저 구조 리뷰와 설계 검토를 요청합니다.
3. 실제 사내 데이터, API Key, 내부 시스템명은 입력하지 않습니다.
4. mcp_call_trace.jsonl이 반드시 생성된다고 단정하지 않습니다.
5. 현재 Day3 예제에서는 State, agent_steps, db_results, rag_results, 결과 JSON/Markdown을 실행 기록 관점으로 해석합니다.
6. tool_registry_fallback.py는 모든 Tool을 완전히 대체하는 구조가 아니라 get_equipment_overview와 search_manual 중심의 단순 fallback 예제입니다.
7. 제한 수정 실습에서는 실제 코드 수정이 아니라 “수정 계획 제안”까지만 요청합니다.

============================================================
실습 마무리 정리
============================================================

오늘 Claude Code 실습의 핵심은 코드를 대신 작성시키는 것이 아닙니다.

Claude Code는 구조를 빠르게 읽고 설계 리뷰 초안을 만드는 도구입니다.
최종 판단은 사람이 해야 합니다.

특히 제조 현업에서는 권한, 반환 데이터 범위, 감사 로그, Guardrail 기준이 중요하기 때문에 AI가 제안한 코드를 바로 반영하지 않고, 변경 범위와 실행 결과를 반드시 사람이 확인해야 합니다.

오늘 확인한 src/day3 구조는 4일차 평가와 Guardrail, 5일차 사내 PoC 전환 전략의 기반이 됩니다.
